**Priority**: MEDIUM
**Assigned to**: developer2
**Description**: Implement an attachments panel that allows users to embed supplementary files within a PDF and to view, download, or remove existing embedded attachments — a standard PDF specification feature used in business, legal, and archival workflows. **Reading existing attachments**: When a PDF is loaded, use pdf-lib's low-level API to inspect the PDF catalog's `/Names` dictionary for an `/EmbeddedFiles` name tree. For each entry, extract the file specification (`/F`, `/UF` for filename, `/Desc` for description) and the embedded file stream (`/EF` → `/F`) including its `/Params` (size, creation date, modification date, MIME type via `/Subtype`). Display these in an "Attachments" side panel (similar to the bookmarks panel) as a list showing filename, file size (formatted human-readable: KB/MB), MIME type icon, and description. **Adding new attachments**: Provide an "Add Attachment" button that opens a file picker (accept all file types). Validate the selected file: enforce a 10MB per-file limit and 50MB total attachments limit, reject executable file types (.exe, .bat, .cmd, .sh, .msi, .dll, .com, .scr) for security. Read the file as an `ArrayBuffer`, then use pdf-lib to embed it: create a file specification dictionary with `/Type /Filespec`, `/F` and `/UF` keys (the filename), `/Desc` (user can optionally enter a description via a small input), and `/EF` pointing to an embedded file stream containing the raw bytes with `/Type /EmbeddedFile`, `/Subtype` set to the MIME type, and `/Params` including `/Size` and `/ModDate`. Add the file specification to the document catalog's `/Names` `/EmbeddedFiles` name tree (create the name tree if it doesn't exist). Update the panel list immediately. **Downloading attachments**: Each attachment in the list has a download button. On click, extract the embedded file stream bytes from the PDF structure, create a `Blob` with the correct MIME type, and trigger a browser download with the original filename. **Removing attachments**: Each attachment has a remove button. On click (with confirmation dialog), remove the file specification entry from the `/EmbeddedFiles` name tree and the associated embedded file stream object. **Saving**: When the user downloads/saves the PDF, all attachment changes (additions and removals) are persisted. Output as `[original-filename]-attachments.pdf` if modifications were made. **UI details**: Add an "Attachments" tab or button in the left sidebar (alongside thumbnails/bookmarks). The panel shows a file count badge. Support drag-and-drop onto the panel to add files. Show a progress indicator for large file embeds. Empty state shows "No attachments — drag files here or click Add." **Keyboard accessibility**: Tab navigation through the attachment list, Enter to download, Delete to remove (with confirmation). ARIA labels on all interactive elements. **Edge cases**: Handle PDFs with no existing `/Names` dictionary (create one), PDFs with existing non-attachment name trees (preserve them), filenames with special characters (sanitize for display but preserve original in PDF), duplicate filenames (append `_2`, `_3`), and zero-byte files (reject with error message). All processing client-side in the browser — no server upload needed.

### TASK-041: Measure and distance tool for PDF pages

**Status**: DONE
**Priority**: MEDIUM
**Assigned to**: developer
**Description**: Build a measurement tool that lets users measure distances, areas, and perimeters on PDF pages — essential for working with architectural drawings, blueprints, floor plans, engineering specs, and technical documents. **Measurement modes**: Implement three measurement tools: (1) **Distance** — click two points on the page to measure the straight-line distance between them, displayed as a line overlay with the measurement value shown at the midpoint. (2) **Perimeter** — click multiple points to form a polyline (double-click or press Enter to finish), showing the total length of all segments and individual segment lengths on hover. (3) **Area** — click multiple points to define a closed polygon (automatically closes on double-click), showing the enclosed area with the polygon outlined in a semi-transparent fill. **Scale calibration**: Provide a "Set Scale" feature where users click two points of a known real-world distance (e.g., a scale bar on a blueprint), then enter the actual measurement (e.g., "10 feet", "5 meters"). The tool calculates a px-to-real-unit ratio and applies it to all subsequent measurements. Support unit selection: points (default/raw), inches, centimeters, millimeters, feet, and meters. Without calibration, measurements default to PDF points (1 point = 1/72 inch). Store the scale factor per page (different pages may have different scales). **Display and interaction**: Render measurements as SVG overlays on the page canvas, using normalized 0–1 coordinates for zoom independence. Each measurement shows: the value label (formatted with 2 decimal places and the unit), colored lines/polygons (blue for distance, green for perimeter, orange for area), and small circular handles at each point. Measurements can be selected (click), deleted (Delete key or X button), and the label can be repositioned by dragging. Provide a measurement list panel (collapsible sidebar or inline panel) showing all measurements grouped by page — each entry shows the type icon, value, and page number. Clicking an entry navigates to that page and highlights the measurement. Include a "Clear all measurements" button and per-measurement delete. **Angle measurement** (bonus): When measuring distance or perimeter, show the angle between segments relative to horizontal (in degrees). **Snapping**: Snap measurement points to page edges, corners, and other measurement points within a 5px tolerance for precision alignment. Show crosshair cursor guides (horizontal and vertical lines extending to page edges) while placing points. **PDF annotation persistence**: Optionally embed measurements into the PDF as line/polygon annotations with measurement markup (`/Subtype /Line` with `/Measure` dictionary or `/Subtype /Polygon`) so they appear in other PDF viewers that support measurement annotations. Use pdf-lib to write the annotation dictionaries. Output as "-measured.pdf". **UI integration**: Add the measure tool as a new section under the "Annotate" tab or as its own top-level tool, with buttons for distance/perimeter/area modes and the scale calibration control. Include a toolbar showing the current scale and selected unit. **Keyboard support**: Escape to cancel the current measurement in progress, Delete to remove selected measurements, Ctrl+Z to undo. All processing client-side in the browser — no additional libraries needed beyond pdf-lib for annotation embedding.
