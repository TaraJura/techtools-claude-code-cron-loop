**Description**: Build a print preparation tool that rearranges PDF pages into optimized layouts for physical printing — covering the three most common print scenarios that standard PDF viewers don't handle well. **(1) N-up layout (multiple pages per sheet)**: Let users place 2, 4, 6, 9, or 16 PDF pages onto a single output page, arranged in a grid. Provide controls for: output page size (A4, Letter, Legal, A3), page order (left-to-right then top-to-bottom, or Z-pattern vs. N-pattern), orientation (auto-detect or force portrait/landscape), spacing between pages (0–20pt gutter), and whether to draw thin border lines around each sub-page for cutting guides. Use pdf-lib to create a new `PDFDocument`, embed each source page as an XObject via `embedPage()`, then draw multiple embedded pages onto each new output page using `page.drawPage()` with calculated x/y offsets and scaling to fit the grid cells. For a document with N pages (pad to nearest multiple of 4 with blank pages), arrange pages in booklet signature order: sheet 1 front = [last, first], sheet 1 back = [second, second-to-last], and so on. Each output page contains two source pages side-by-side on a landscape sheet. Users select the paper size and can enable "creep compensation" — a slight inward shift of inner pages to account for paper thickness in thick booklets (configurable 0–5pt per sheet). Show a visual preview of the folded booklet page order so users can verify correctness before generating. **(3) Poster / tile printing**: Split a single PDF page across multiple sheets for large-format output. Users specify the output paper size and the desired scale factor (e.g., 200%, 400%, or "fit to X×Y sheets"). The tool divides the source page into a grid of tiles, each the size of the output paper minus configurable overlap margins (5–25mm, default 15mm) so tiles can be trimmed and aligned. Draw crop marks and alignment crosses at tile corners. Number each tile (e.g., "Tile 2 of 6 — Row 1, Col 2") in a small label in the margin area. **Live preview**: Show a real-time visual preview of the output layout as settings change. For n-up, render a thumbnail grid showing how pages will be arranged. For booklet, show the sheet order with page numbers. For poster, show the tile grid overlaid on the source page with dashed lines indicating cuts. **UI integration**: Add a "Print Prep" tool under the Pages tab or as a new top-level tab. The panel has three sub-tabs: "N-up", "Booklet", and "Poster". Each shows the relevant controls and a preview area. Include a page range selector (process all pages or a custom range). **Output**: Save as `[original-filename]-nup.pdf`, `-booklet.pdf`, or `-poster.pdf` respectively. Show a progress bar during generation for large documents. **Edge cases**: Handle PDFs with mixed page sizes (scale each source page individually to fit the target grid cell), single-page PDFs in booklet mode (pad with 3 blank pages), very large PDFs in n-up mode (process in batches of 50 pages to avoid memory issues, using `setTimeout` between batches), and rotated source pages (detect `/Rotate` and compensate). All processing client-side in the browser using pdf-lib and pdf.js for preview rendering.

This is especially useful for scanned documents with irregular margins. **Implementation with pdf-lib**: PDF pages have multiple boundary boxes defined in their page dictionary: `/MediaBox` (physical page size), `/CropBox` (visible area — this is what viewers display), `/TrimBox` (intended finished page), and `/BleedBox` (printing bleed area). For cropping, set the `/CropBox` on the target page to the user-defined rectangle coordinates, converted from screen coordinates to PDF coordinates (flip Y-axis, scale by zoom factor). When applying to all pages, use the same crop dimensions relative to each page's own size — useful for batch-trimming a scanned book where all pages have similar excess margins. For pages with different sizes, calculate the crop proportionally or let the user choose between "absolute dimensions" (same crop box size) and "relative margins" (same amount trimmed from each edge). **Aspect ratio constraints**: Provide optional aspect ratio locking — common presets like A4 (1:√2), Letter (8.5:11), Square (1:1), 16:9, and custom ratio. When locked, adjusting one dimension auto-adjusts the other. **Undo support**: Push the pre-crop page state onto the existing undo history stack (from the pages module's `pushHistory()`) so Ctrl+Z reverts the crop. This is critical — the tester will verify undo works. **Preview**: Show the cropped result as a real-time preview before applying. Include a toggle to switch between "preview cropped" and "show original with crop overlay" views. **Reset**: Provide per-page and "reset all" buttons that restore the original CropBox (or remove it to fall back to MediaBox). **UI integration**: Add a "Crop" tool as a new sub-panel under the existing "Pages" tab, alongside the existing page management tools. Include a toolbar button with a crop icon for quick access. When crop mode is active, the cursor changes to a crosshair on the page canvas. **Output**: Save as `[original-filename]-cropped.pdf`. **Edge cases**: Handle pages that already have a CropBox different from MediaBox (show the existing crop and allow adjustment), rotated pages (apply crop in the page's coordinate space respecting `/Rotate`), pages with different sizes in the same document (show per-page dimensions in the numeric inputs), very small crop areas (minimum 50×50pt to avoid unusable pages), and pages where auto-crop finds no whitespace margins (show "No whitespace margins detected" message). ### TASK-050: Headers, footers, and page numbering tool

**Priority**: HIGH
**Assigned to**: developer
**Description**: Build a headers and footers tool that lets users add recurring text elements — page numbers, document title, date, author, and custom text — to every page (or a page range) of a PDF. This is one of the most commonly requested PDF editing features, essential for professional documents, reports, legal filings, and academic papers. **Header/footer zones**: Divide the top and bottom margins of each page into three zones each: left, center, and right — giving 6 configurable slots total (top-left, top-center, top-right, bottom-left, bottom-center, bottom-right). Each zone can contain a combination of static text and dynamic tokens. **Dynamic tokens**: Support placeholder tokens that auto-fill per page: `{page}` (current page number), `{pages}` (total page count), `{date}` (current date in configurable format: MM/DD/YYYY, DD/MM/YYYY, YYYY-MM-DD, or a custom date string), `{filename}` (the PDF filename without extension), and `{title}` (PDF document title from metadata, or filename if no title). Allow combining tokens with text, e.g., "Page {page} of {pages}" or "Confidential — {date}". **Page numbering styles**: Support multiple numbering formats: Arabic (1, 2, 3), Roman lowercase (i, ii, iii), Roman uppercase (I, II, III), alphabetic lowercase (a, b, c), and alphabetic uppercase (A, B, C). Allow a starting page number offset (e.g., start at page 3 to account for a cover page and table of contents). Provide a "Skip first page" toggle (common for documents with a cover page that shouldn't show a header/footer) and a configurable page range (e.g., apply to pages 5–50 only). Support different formatting for odd/even pages — e.g., page number on the right for odd pages and left for even pages (book-style alternating). **Font and style controls**: Font family (Helvetica, Times, Courier — standard PDF fonts), font size (6–24pt, default 10pt), font color (black default, with color picker), bold/italic toggles, and opacity (0.1–1.0). Provide separate styling per zone if the user enables "Advanced mode" — otherwise a single style applies to all zones. **Margin and positioning**: Configurable margin offsets — how far the header is from the top edge and the footer from the bottom edge (default 36pt / 0.5 inch). Horizontal margins from left/right edges (default 36pt). Show visual margin guides on the preview. **Separator line**: Optional thin horizontal line between the header and page content (and between page content and footer). Configurable line thickness (0.5–3pt), color, and style (solid, dashed). **Live preview**: Show a real-time preview of the header/footer on the current page as the user adjusts settings. Render the text overlays on the PDF viewer canvas, updating on every input change with 50ms debounce. Include a page navigator in the preview to verify how the headers/footers look on different pages (especially useful when odd/even alternating is enabled). **Implementation with pdf-lib**: Load the PDF via `PDFDocument.load()`. For each target page, resolve dynamic tokens by substituting `{page}` with the calculated page number (respecting the starting offset), `{pages}` with the total page count of the target range, `{date}` with the formatted current date, and `{filename}`/`{title}` from the document's info dictionary or the loaded filename. Embed the chosen font via `pdfDoc.embedFont(StandardFonts.Helvetica)` etc. For each zone, calculate the text position: top zones use `y = page.getHeight() - topMargin`, bottom zones use `y = bottomMargin - fontSize`. Left zones use `x = horizontalMargin`, center zones use `x = (page.getWidth() - textWidth) / 2`, right zones use `x = page.getWidth() - horizontalMargin - textWidth`. Compute text width using `font.widthOfTextAtSize(text, fontSize)`. Draw with `page.drawText()` and set color using `rgb()` from pdf-lib. For separator lines, use `page.drawLine()` spanning the page width between the horizontal margins.
