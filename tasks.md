### TASK-143: Document mail merge — generate batch personalized PDFs from a template and CSV/JSON data source

**Status**: DONE
**Priority**: MEDIUM
**Assigned to**: developer2
**Description**: Build a document mail merge tool that takes a single PDF template containing placeholder variables and a CSV or JSON data source, then generates a batch of personalized PDF documents — one per data row. Add a new tool tab "Mail Merge" (icon: document with list/rows, suggesting template + data). Features: (1) **Template variable syntax** — define placeholders in the PDF using a `{{variable_name}}` syntax. Users place these as text overlays via text-overlay.js or within form fields. The mail merge engine scans all text content, text overlays, form fields, and stamp text for `{{...}}` patterns. Support dot notation for nested JSON data (e.g., `{{address.city}}`). (2) **Data source import** — accept CSV files (parsed client-side using a lightweight CSV parser — handle quoted fields, commas in values, UTF-8 BOM) or JSON files (array of objects). Show a data preview table after import displaying the first 5 rows and all detected column headers. Auto-map column headers to detected template variables by exact name match, with a manual mapping UI for mismatches (dropdown per variable to select which column it maps to). Display unmatched variables and unused columns as warnings. (3) **Variable insertion helper** — a panel listing all column headers from the imported data, with "Insert" buttons next to each that place a `{{variable}}` text overlay at the current cursor position on the PDF page, making template creation easy without manual typing. (4) **Preview mode** — after mapping, show a live preview of the merged document for any row. A row selector (prev/next arrows + row number input) lets the user scrub through all records seeing exactly how each output will look. Render the preview by cloning the current PDF state and performing text replacement in real-time. (5) **Merge engine** — for each data row: clone the template PDF bytes (via pdf-lib), iterate all text overlays and replace `{{variable}}` patterns with the row's values, fill matching form fields programmatically, replace placeholder text in stamp annotations, and handle missing values gracefully (configurable: leave blank, show "N/A", or skip the placeholder). Support basic formatting directives: `{{date:iso}}` for ISO date formatting, `{{amount:currency}}` for number formatting with locale currency symbol, and `{{name:upper}}` / `{{name:lower}}` for case transforms. (6) **Output options** — three output modes: (a) single merged PDF with all personalized pages concatenated (with optional page break between each record), (b) individual PDF files per row (downloaded as a ZIP via JSZip), or (c) individual PDFs with configurable filenames using variables (e.g., `Invoice_{{invoice_number}}_{{name}}.pdf`). (7) **Batch generation with progress** — process rows with a progress bar ("Generating document 45 of 200…"). Use requestAnimationFrame or setTimeout batching to keep the UI responsive. Show estimated time remaining for large batches. Allow cancellation mid-batch (produces partial output of completed documents). (8) **Conditional content** — basic conditional logic: `{{#if variable}}content{{/if}}` to include content only when a variable has a value, and `{{#ifnot variable}}fallback{{/ifnot}}` for the inverse. (9) **Row filtering** — before generating, allow filtering which rows to include: all rows, a numeric range (rows 1–50), or a filter expression (e.g., `status == "active"`). (11) **Error handling** — validate the data source before merging: flag rows with missing required variables, detect type mismatches (e.g., date formatting on non-date values), and display a validation report. Allow the user to fix data issues or exclude problematic rows before proceeding. Register with the action registry (actions: `mail-merge.open`, `mail-merge.preview`, `mail-merge.generate`) and event bus (events: `mail-merge:data-loaded`, `mail-merge:preview-row`, `mail-merge:generation-start`, `mail-merge:generation-complete`). Keyboard shortcut: `Ctrl+Shift+M` (if available; otherwise `Ctrl+Alt+M`) to open the mail merge panel.
