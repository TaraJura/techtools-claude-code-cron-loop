## 2026-04-05

- [TASK-069] **NEW FEATURE** — Page resize and custom dimensions tool. New module `js/page-resize.js` adds a "Resize" button to the Pages toolbar that opens a dedicated resize panel. **Standard sizes**: Dropdown with A4, US Letter, Legal, A3, A5, B5, Tabloid, Executive, and Custom. Displays dimensions in the user's preferred unit (mm, inches, or points) with real-time conversion. Custom size inputs with min/max validation. **Orientation**: Portrait/landscape toggle that swaps width and height. **Content handling modes**: Four strategies — Scale to Fit (uniform scale, centered, preserving aspect ratio), Stretch to Fill (non-uniform, with distortion warning), Center No Scale (original size centered on new page), and Reposition (manual X/Y offset). **Page scope**: Apply to current page, all pages, custom range (e.g. "1-5, 8, 12-20"), or pages matching a specific detected size. Summary shows affected page count. Mixed-size document detection with independent scaling per page. **Margins**: Collapsible Advanced section with top/right/bottom/left margin inputs, link toggle for uniform margins, all in the user's selected unit. **Print production**: Collapsible Bleed section with configurable bleed amount and trim marks drawn at page corners. **Live preview**: Canvas preview showing the resized page with content placement, original page outline (dashed), new page border (solid), margin guides, and bleed area. Before/after toggle. Scale percentage display. 100ms debounce on parameter changes. **Implementation**: Uses pdf-lib's `embedPage()` and `drawPage()` to transfer all page content as a single XObject, preserving text, images, vector graphics, and annotations. Creates new pages with target dimensions, draws embedded original content with calculated placement, removes originals. Handles rotated pages, zero-dimension pages, and extreme sizes. Outputs `[filename]-resized.pdf`. All processing client-side.

- [TASK-068] **NEW FEATURE** — Text-to-speech PDF reader with synchronized word highlighting. New module `js/tts.js` implements a full TTS reading system using the Web Speech API. **Speech controls**: Floating control bar at the bottom of the viewer with play/pause, stop, skip forward/back by sentence, speed slider (0.5x–3x), volume slider, voice selector (grouped by language), and reading mode selector. **Text extraction**: Extracts text from PDF pages via pdf.js `getTextContent()`, sorts in reading order (top-to-bottom, left-to-right), handles ligatures (fi, fl, ff, ffi, ffl), soft hyphens, and multi-column layouts. Optional header/footer skipping. Caches extracted text per page. **Synchronized highlighting**: Canvas overlay on the PDF page highlights the current word or sentence in real-time using `SpeechSynthesisUtterance.onboundary` events. Configurable highlight color/opacity and word-level or sentence-level granularity. Auto-scrolls to keep highlighted text visible. **Reading modes**: Page mode (reads one page, then pauses), Continuous mode (reads through entire document with auto page advance), and Selection mode (reads user-selected text). **Chrome compatibility**: Breaks long text into ~200-char chunks at sentence/word boundaries to work around Chrome's utterance timeout bug, with 50ms gap between chunks. **Language detection**: Auto-detects document language from text content (supports EN, DE, FR, ES, ZH, JA, KO, RU, AR) and selects matching voice. **Tab visibility**: Auto-pauses when user switches tabs, resumes on return. **Progress tracking**: Page progress bar, sentence counter, and integration with reading progress tracker. **Preferences**: All settings (speed, volume, voice, highlight style, reading mode) persisted in localStorage. **Command palette**: Registered as "Read Aloud", "Pause Reading", "Stop Reading", "Increase/Decrease Reading Speed". Toolbar button with speaker icon added to viewer toolbar. All client-side — no server involvement.

- [TASK-066] **NEW FEATURE** — Undo/redo editing history with visual operation timeline. New module `js/undo-redo.js` implements a comprehensive Command pattern-based undo/redo system. **Command classes**: 15 concrete command types covering annotations (add/delete/move/edit), page operations (reorder/delete/rotate/insert/crop), text overlays, watermarks, form fields, drawings, stamps, signatures, and batch operations. `GenericCommand` for simple execute/undo pairs. `BatchCommand` groups multiple operations into a single undo entry. **UndoRedoManager**: Singleton manager with undo/redo stacks, configurable max history (100 ops), multi-step undo/redo-to-index, and event-driven state notifications via EventBus. `execute()` and `pushExecuted()` methods for integration with existing modules. **Toolbar buttons**: Undo/redo buttons added to main viewer toolbar with disabled state management and descriptive tooltips. Dropdown arrow next to undo shows last 10 operations for multi-step undo. **History sidebar panel**: New "History" tab in the sidebar (clock icon) with vertical timeline showing all operations chronologically. Each entry shows operation icon, description, relative timestamp, and clickable page badges for navigation. Current state divider separates undo (active) from redo (dimmed) entries. Click any entry to jump to that state. Counter shows operations used vs max. Clear button with confirmation. **Keyboard shortcuts**: Ctrl+Z for undo, Ctrl+Shift+Z/Ctrl+Y for redo. Skips interception when text inputs are focused. **Command palette**: Registered as "Undo", "Redo", "Show History Panel", "Clear History" actions. History auto-clears on new file load. All client-side.

- [TASK-065] **NEW FEATURE** — Digital signature verification and certificate inspection tool. New module `js/verify-signatures.js` adds a "Verify Signatures" sidebar panel (shield icon with checkmark). **Signature detection**: Scans raw PDF bytes for `/ByteRange` entries to locate all digital signature fields. Extracts signer name (`/Name`), date (`/M`), reason, location, contact info, filter, and sub-filter from the signature dictionary. **PKCS#7/CMS parsing**: Built-in ASN.1/DER parser decodes the PKCS#7 SignedData structure from the `/Contents` hex string. Extracts embedded X.509 certificates, digest algorithm, signature algorithm, signed attributes (signingTime, messageDigest), and unsigned attributes (timestamp tokens). **Integrity verification**: Computes hash (SHA-256/384/512/SHA-1) over the byte ranges covered by the signature using Web Crypto API, compares against the messageDigest in CMS signed attributes. Reports whether document has been modified since signing. Detects uncovered bytes after signature (potential shadow attacks). **Certificate inspection**: Modal displays full certificate details — subject, issuer, serial number, validity period (with expired/valid badge), public key algorithm and size, key usage extensions, certificate chain visualization as interactive tree. SHA-256 fingerprint computed via Web Crypto. **Timestamp detection**: Identifies RFC 3161 timestamp tokens in unsigned attributes, extracts TSA certificate name. **Status indicators**: Color-coded entries (green=valid, yellow=warning, red=invalid, gray=unknown) with shield icons. Expandable detail rows showing verification steps. Sidebar badge shows signature count. **Export report**: Generates downloadable TXT verification report with all signature details, certificate info, and integrity results. **Command palette**: Registered as "Verify Signatures" and "Check Document Integrity" actions. All processing client-side using Web Crypto API — no PDF data leaves the browser.
