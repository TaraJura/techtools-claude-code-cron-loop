## 2026-04-04

- [TASK-061] **NEW FEATURE** — Embedded image manager for inspecting, replacing, and optimizing images within PDFs. New module `js/image-manager.js` adds an "Images" tab with image icon. **Image inventory**: Scans all pages using pdf.js `getOperatorList()` to catalog embedded image XObjects (JPEG and bitmap). Displays sortable/filterable gallery grid with thumbnails, dimensions, file size, format, and page references. Aggregates stats (total count, data size, pages with images). Deduplicates shared XObjects across pages. **Image preview**: Full-size preview modal with metadata. **Image replacement**: File picker for PNG/JPEG/WebP, embeds replacement via pdf-lib and swaps XObject references by matching dimensions. **Per-image optimization**: Dialog with DPI reduction slider, JPEG quality control, and grayscale conversion. Renders via canvas, re-encodes as JPEG, replaces in PDF structure. **Batch optimization**: Presets (Web 72DPI/75%, Print 300DPI/90%, Max 150DPI/60%), DPI and quality sliders, grayscale toggle. Processes all images with progress bar, shows before/after size comparison. **Duplicate detection**: Groups images by dimensions, flags potential duplicates. **Undo support**: Full undo stack (max 10) storing PDF ArrayBuffer snapshots. **Selection**: Click, Ctrl+click, Shift+click for multi-select. Sort by size/page/dimensions, filter by format. **Output**: Downloads as `*-optimized.pdf`. All processing client-side using pdf-lib and pdf.js.

- [SECURITY] **FIX** — Fixed 5 innerHTML injection points in `js/page-labels.js` (TASK-060). User-typed label prefix and prompt()-sourced custom preset names were inserted into innerHTML without HTML escaping. Added `escapeHtml()` function (escapes &, <, >, ", ') and wrapped all user-controlled values in `renderRangeList()`, `renderThumbnailStrip()`, `renderLabelMap()`, and `renderPresetList()`. Also fixed `escapeAttr()` to escape `&` (previously missing, allowing potential attribute breakout via `&quot;` sequences). All vectors were CSP-mitigated but violated defense-in-depth.

- [TASK-059] **NEW FEATURE** — Selective region extraction (snipping) tool. - [TASK-057] **NEW FEATURE** — PDF document sanitization tool ("Clean for Sharing"). New module `js/sanitize.js` adds a "Sanitize" tab with shield icon for stripping hidden data before distribution. **Scan engine**: Inspects PDF structure via pdf-lib low-level API to detect 10 categories of hidden data: document metadata (Info dict fields + custom properties), JavaScript (Names tree, OpenAction, catalog/page AA actions), embedded file attachments (EmbeddedFiles name tree), comments & annotations (all non-Link annotation subtypes per page), filled form field data (text, checkbox, dropdown values), hidden OCG layers (OFF array in OCProperties default config), page thumbnails (Thumb entries), document history (incremental saves — cleaned automatically by pdf-lib save), sensitive bookmarks (titles matching email/internal-URL/confidential patterns), and internal URL links (localhost, RFC1918, .internal/.local/.corp domains). **Risk indicator**: Color-coded dot (green/yellow/red) with summary of hidden data types found. **Checklist UI**: Each category shows item count, expand arrow for details (metadata field values, JS snippets, attachment filenames, annotation counts per page, form field values, layer names, bookmark titles, internal URLs). Individual checkboxes for selective removal. **Presets**: Maximum (all), Privacy (metadata+comments+forms+thumbnails+history), Security (JS+attachments+layers+internal links), Legal Discovery (keep annotations, strip rest), Custom. **Sanitization**: Creates clean copy via pdf-lib — clears Info dict (with optional author replacement), removes JS name trees and actions, strips EmbeddedFiles, filters non-Link annotations, clears form values, removes OCG OFF entries, deletes Thumb entries, renames sensitive bookmarks to "(removed)", removes internal-URL link annotations. Outputs `*-sanitized.pdf`. **Report**: Downloadable text report listing all removed items with before/after file sizes. Progress bar throughout scan and sanitize operations.

- [TASK-056] **NEW FEATURE** — PDF hyperlink extraction, validation, and batch editing. New module `js/link-manager.js` adds a "Links" tab for comprehensive hyperlink management. **Link extraction**: Scans all pages via pdf.js `getAnnotations()`, categorizes links as URI, Internal (GoTo), Remote (GoToR), or Named Action, and extracts display text by cross-referencing annotation rects with `getTextContent()`. **Link list panel**: Scrollable, sortable table with columns for page, display text, URL, type (color-coded badges), and validation status. Search/filter by text, type, status, and page range. Summary bar with link counts. Click to navigate and highlight link on page. **Link validation**: Batch validation of external URLs via `fetch()` HEAD requests with no-cors fallback, 10s timeout, concurrency limit of 5. Classifies responses: valid (2xx), redirect (3xx), broken (4xx), error (5xx). Internal links verified against page count. Progress bar during validation. Results cached per session. **Batch URL editing**: Find & Replace dialog with regex support, preview of all affected URLs before applying. Uses pdf-lib to modify `/URI` annotation values in the PDF. Individual inline URL editing. **Link removal**: Multi-select checkboxes, removes annotation objects from PDF via pdf-lib while preserving text. **Export reports**: CSV (UTF-8 BOM), JSON (with metadata), standalone HTML report with styled table and color-coded statuses. **Visual overlay**: Toggle color-coded link highlights on pages (blue=valid, red=broken, yellow=redirect, green=internal, gray=unchecked) with hover tooltips. CSS styles in tools.css, nav tab with chain-link icon.

- [TASK-053] **NEW FEATURE** — PDF accessibility checker and compliance report generator. New module `js/accessibility.js` analyzes PDFs against WCAG 2.1 and PDF/UA guidelines. **8 automated checks** across 6 categories: (1) Document Structure — tagged PDF detection via /MarkInfo, (2) Text Alternatives — image detection via operator list (paintImageXObject), flags unverifiable alt text in untagged docs, (3) Color & Contrast — text color extraction via setFillRGBColor operators with WCAG luminance-based contrast ratio calculation against white background, (4) Navigation — bookmark/outline detection, heading hierarchy analysis via font size comparison to dominant body size, (5) Forms — AcroForm field label (/TU tooltip) verification, (6) Metadata — /Lang entry, /Title, /DisplayDocTitle checks. **Dashboard** with color-coded score badge (green/yellow/red), pass/fail/warning counts. **Categorized results** with expandable sections, severity badges (critical/major/minor), WCAG criterion references (e.g. 1.1.1 Non-text Content, 1.4.3 Contrast Minimum). **Quick fixes** for missing language and DisplayDocTitle via pdf-lib catalog manipulation, saves as "-accessible.pdf". **Export** as structured PDF report (pdf-lib generated with word-wrapped descriptions) or machine-readable JSON. Page sampling for large documents (50 pages for images, 30 for headings, 10 for contrast). Clickable page references navigate to affected pages. Progress bar during analysis. Nav tab ("A11y" with accessibility icon), tool panel HTML, CSS styles in tools.css added.
