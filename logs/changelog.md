## 2026-04-04

- [TASK-053] **NEW FEATURE** — PDF accessibility checker and compliance report generator. New module `js/accessibility.js` analyzes PDFs against WCAG 2.1 and PDF/UA guidelines. **8 automated checks** across 6 categories: (1) Document Structure — tagged PDF detection via /MarkInfo, (2) Text Alternatives — image detection via operator list (paintImageXObject), flags unverifiable alt text in untagged docs, (3) Color & Contrast — text color extraction via setFillRGBColor operators with WCAG luminance-based contrast ratio calculation against white background, (4) Navigation — bookmark/outline detection, heading hierarchy analysis via font size comparison to dominant body size, (5) Forms — AcroForm field label (/TU tooltip) verification, (6) Metadata — /Lang entry, /Title, /DisplayDocTitle checks. **Dashboard** with color-coded score badge (green/yellow/red), pass/fail/warning counts. **Categorized results** with expandable sections, severity badges (critical/major/minor), WCAG criterion references (e.g. 1.1.1 Non-text Content, 1.4.3 Contrast Minimum). **Quick fixes** for missing language and DisplayDocTitle via pdf-lib catalog manipulation, saves as "-accessible.pdf". **Export** as structured PDF report (pdf-lib generated with word-wrapped descriptions) or machine-readable JSON. Page sampling for large documents (50 pages for images, 30 for headings, 10 for contrast). Clickable page references navigate to affected pages. Progress bar during analysis. Nav tab ("A11y" with accessibility icon), tool panel HTML, CSS styles in tools.css added.

## 2026-04-03

- [TASK-034] **BUG FIX** — Hyperlink internal page navigation was broken. The `navigate:page` event emitted by `hyperlinks.js` (and `autosave.js`) had no listener. Added `bus.on('navigate:page', (pageNum) => goToPage(pageNum))` in `viewer.js` init function to wire up page navigation for internal PDF links, link list panel clicks, and autosave session restore.
- [TASK-050] **NEW FEATURE** — Headers, footers, and page numbering tool. New module `js/headers-footers.js` adds a comprehensive header/footer editor. **6 configurable zones** (top-left, top-center, top-right, bottom-left, bottom-center, bottom-right) with visual grid layout matching page positions. **Dynamic tokens**: `{page}`, `{pages}`, `{date}`, `{filename}`, `{title}` with one-click insertion buttons per zone. **5 numbering styles**: Arabic, Roman lowercase/uppercase, alphabetic lowercase/uppercase. **Page controls**: start number offset, skip first page toggle, custom page range (e.g. "2-10, 15"), odd/even page alternation (book-style mirroring of left/right zones). **Typography**: Helvetica/Times/Courier with bold/italic variants (all 12 pdf-lib StandardFonts), font size 6-24pt, color picker, opacity slider. **Margins**: configurable top/bottom/horiz margins with visual margin guides in preview. **Separator lines**: optional horizontal rules between header/footer and page content. **Date formats**: YYYY-MM-DD, MM/DD/YYYY, DD/MM/YYYY, MMM DD YYYY. **5 built-in presets**: "Page numbers bottom center", "Title & date header", "Legal footer", "Book style", "Academic paper". Custom presets save to localStorage. **Live preview** with debounced canvas rendering, page navigator for verifying different pages. **Remove existing headers/footers** mode using pdf.js text extraction + pdf-lib white-rectangle redaction. Text truncation with ellipsis for overflow. All processing client-side via pdf-lib. Nav tab, tool panel HTML, CSS styles added.
- [SECURITY] **XSS FIX** — `templates.js` line 688: custom template names from JSON imports were used unescaped in `aria-label` HTML attribute, allowing attribute injection via crafted import files. Fixed by wrapping with `escHtml()`. Also hardened import validation: `templateId` now validated against known generators whitelist, name truncated to 100 chars, strict type checks on all imported fields. CSP mitigated script execution risk but DOM injection was possible.
- [TASK-049] **NEW FEATURE** — Pre-built document templates system. New module `js/templates.js` generates PDFs entirely client-side using pdf-lib. 12 built-in templates across 4 categories: **Basic** (Blank Page, Lined Paper, Graph Paper, Dot Grid), **Education** (Cornell Notes, Sheet Music), **Business** (Invoice, Letterhead, Certificate, Meeting Agenda), **Productivity** (To-Do List, Weekly Planner). Each template opens a configurator with common options (page size A4/Letter/Legal/A3, portrait/landscape, page count 1-500, margin, color theme classic/blue/sepia/dark) plus template-specific options (line spacing, grid size, stave count, certificate type, checkbox rows, planner hours, etc.). Live canvas preview at 50% scale updates as settings change. Batched multi-page generation with progress bar for >10 pages. Custom templates saved to localStorage with rename/delete and JSON import/export. Welcome screen shows 4 popular template quick-start cards. Generated PDFs load directly into the viewer for immediate editing. Templates nav tab, tool panel HTML, and CSS added.
- [TASK-048] **NEW FEATURE** — Command palette and customizable keyboard shortcuts. Three new modules: `js/action-registry.js` (central registry for all app actions with unique IDs, categories, descriptions, default shortcuts, and execute callbacks), `js/command-palette.js` (Ctrl+K opens a modal overlay with fuzzy search across all registered actions — shows recent actions, category headers, match highlighting, keyboard navigation with arrow keys/Enter/Escape), `js/keyboard-shortcuts.js` (global keydown handler dispatches registered shortcuts, customization panel with per-action edit/capture mode, conflict detection, export/import JSON, reset to defaults, printable cheat sheet). ~50 built-in actions registered across File, View, Page, Annotate, Tools, Convert, and Settings categories. Custom bindings persist to localStorage. Command palette button added to header. Browser-reserved shortcuts (Ctrl+T/W/N etc.) blocked from capture. Full ARIA accessibility (dialog, listbox, activedescendant).
- [TASK-047] **ENHANCED FEATURE** — Page crop and trim tool. Extended the existing crop module with: aspect ratio constraints (A4, Letter, Square, 16:9, custom W:H), extended apply modes (all pages, current page, even/odd pages, custom page range), "Also resize MediaBox" checkbox for irreversible size reduction, undo integration via pages module's `pages:save-undo` event (Ctrl+Z reverts crops), rotated page handling (adjusts crop coordinates for 90°/270° rotated pages), minimum 50×50pt crop area enforcement, and "No whitespace margins detected" message for auto-crop. Updated: `js/crop.js`, `index.html` (crop panel HTML), `css/tools.css` (new control styles).
- [TASK-024] **BUG FIX** — Added undo support for page insertion. `insert-pages.js` now emits `pages:save-undo` before applying insertions. `pages.js` saves pre-insertion state (page items + PDF bytes) onto the undo history stack. Ctrl+Z fully reverts insertions by restoring both page metadata and the original PDF document via pdf.js reload. Also added `_skipPdfReady` flag to prevent `onPdfReady` from rebuilding state during undo restore.
- [SECURITY] **XSS FIX** — `qrbarcode.js` line 682: user-supplied QR/barcode data (`p.codeData`) was interpolated into `innerHTML` via template literal without HTML escaping in the placement list UI. Added `escapeHtml()` function and applied it to sanitize the output. CSP `script-src 'self'` mitigated inline script execution, but the raw HTML injection was still a risk for UI manipulation.
- [TASK-041] **NEW FEATURE** — Measure & distance tool for PDF pages. Three measurement modes: Distance (click two points for straight-line measurement with angle display), Perimeter (click multiple points to form a polyline, double-click or Enter to finish), and Area (click to define a closed polygon with semi-transparent fill). Scale calibration system: click two known-distance points, enter real-world measurement with unit selection (points, inches, cm, mm, feet, meters) — stored per-page. SVG overlays rendered in normalized 0–1 coordinates for zoom independence, with colored lines/polygons (blue/green/orange) and circular handles. Point snapping to page edges, corners, and other measurement points within 5px tolerance. Crosshair cursor guides while placing points. Measurement list panel grouped by page with click-to-navigate and per-item delete. Undo history (30 deep), keyboard shortcuts (Escape cancel, Delete remove, Ctrl+Z undo, Enter finish). PDF annotation persistence via pdf-lib — embeds measurement lines, polygons, circles, and value labels into the PDF, output as `-measured.pdf`. New file: `js/measure.js`. Nav tab, panel HTML, and CSS added.

## 2026-04-02

- [TASK-039] **NEW FEATURE** — Interactive PDF form creation tool: enables users to author form fields on any PDF page. Adds a "Create Form" mode within the Forms tab with five field types (text input, checkbox, radio button, dropdown select, push button). Users click-and-drag on the PDF canvas to draw field bounding rectangles with live blue dashed preview. Placed fields rendered as semi-transparent colored overlays (blue/green/orange/purple/gray by type) with field name labels. Selection with 8-point resize handles, multi-select via Ctrl+click and box-select. Properties panel for editing field name, position, size, font size, options, required/read-only flags. Snapping to page edges, other field edges, and centers with red guide lines. Alignment toolbar (left/right/top/bottom/center/distribute). Keyboard shortcuts: Delete, Ctrl+D duplicate, Ctrl+Z/Y undo/redo (50-entry history), arrow nudge (1pt/10pt with Shift), Tab to cycle, Ctrl+A select all. Imports existing form fields from loaded PDFs. Generates form-enabled PDF via pdf-lib with proper AcroField types (PDFTextField, PDFCheckBox, PDFRadioGroup, PDFDropdown, PDFButton) and NeedAppearances flag. Outputs as `[filename]-form.pdf`. New file: `js/form-creator.js`. CSS added to `tools.css`.
- [TASK-013] **BUG FIX** — Watermark "Below content" layer option now works correctly. Replaced non-functional `drawTextBelow()`/`drawImageBelow()` stubs (which just called the same above-content draw methods) with `reorderContentBelow()`, which uses pdf-lib's `PDFName` API to reorder the page's `Contents` array so the watermark content stream is placed before existing page content streams. PDF renders streams in array order, so the watermark now correctly appears behind existing content when "Below content (background)" is selected.
- [TASK-037] **NEW FEATURE** — Table extraction tool: detect tabular data in PDF pages and export as CSV, XLSX, or copy to clipboard. Grid-based detection clusters text items by Y-coordinate into rows, identifies column boundaries via X-position analysis, and builds 2D cell grids. Supports scanning current page, all pages, or custom ranges with progress bar. Detected tables shown as clickable result cards with data preview (first 3 rows). Full editable HTML table preview for correcting cells before export. Manual region selection fallback — draw rectangle on PDF page with optional column count override. CSV export with UTF-8 BOM for Excel compatibility. XLSX export via SheetJS with auto-column-width and number detection. Tab-separated clipboard copy. Options: header row detection, cell trimming, row tolerance slider. Added as "Tables" sub-tab under Convert panel. SheetJS (xlsx.full.min.js) added to lib/.
- [TASK-034] **NEW FEATURE** — Hyperlink annotations tool: add, edit, detect, and navigate links in PDFs. Detects existing `/Link` annotations from loaded PDFs via pdf.js and renders them as interactive overlays (blue for URL, green for internal page, orange for email). Users can click-and-drag to create link regions with a dialog for URL/page/email input. Double-click to edit, right-click context menu for edit/copy/delete. "Show All" toggle highlights all links with colored overlays. Link list sidebar panel organizes all links by page with click-to-navigate. Undo/redo support (Ctrl+Z/Ctrl+Shift+Z). PDF export embeds links as proper `/Link` annotations with `/URI` and `/GoTo` action dictionaries via pdf-lib. Output as `-linked.pdf` when only hyperlinks are modified.
- [TASK-033] **NEW FEATURE** — Continuous scroll & dual-page spread view modes for the PDF viewer. Added a 3-button view mode selector to the toolbar (single page / continuous scroll / dual-page spread). Continuous scroll mode renders all pages vertically with IntersectionObserver-based lazy loading (renders pages near viewport, unloads distant ones to conserve memory), updates page indicator on scroll via requestAnimationFrame. Dual-page spread mode displays side-by-side page pairs with a cover-page toggle (page 1 alone vs paired). Both modes integrate with existing zoom, navigation, thumbnails, keyboard shortcuts, and all overlay modules (annotations, search, etc.). View mode and cover preference persisted in localStorage. Emits `viewmode:change` event on the bus for future module adaptation.
- [TASK-031] **NEW FEATURE** — Extract Images tool: added a third sub-tab under Convert that scans PDF pages for embedded images using pdf.js operator lists (paintImageXObject / paintJpegImageXObject). Displays found images in a gallery grid with thumbnails, dimensions, page number, and file size. Supports click/Shift+click/Ctrl+A selection, PNG/JPG export format with quality slider, single-image direct download, multi-image ZIP bundling with page-based subfolder structure via JSZip, clipboard copy, progress bars, large image warnings (>10MB), and empty state for imageless PDFs. JPEG images are extracted as raw bytes to avoid quality loss.

## 2026-04-01

- [TASK-016] **SECURITY FIX** — Redaction tool: rewrote `applyRedactions()` to use page-flattening instead of merely drawing opaque rectangles over content. Redacted pages are now rendered to a canvas via pdf.js, redaction rectangles are drawn on top (physically destroying pixel data), and the flattened raster image replaces the original page in the output PDF via pdf-lib. This guarantees no underlying text, vector, or image data survives. Unredacted pages are copied as-is to preserve quality. Updated confirm dialog to accurately describe the flattening process. Removed unused `rgb` import.
- [TASK-020] Fixed crop tool bugs: (1) CRITICAL — changed `[...mb]`/`[...cb]` array spread to `{ ...mb }`/`{ ...cb }` object spread on line 684, fixing `TypeError: mb is not iterable` crash in applyCropAndDownload(). (2) Implemented proper undo that rebuilds the PDF with original mediaBox/cropBox values and downloads the uncropped version, replacing the previous no-op that only reset the UI overlay.
- [TASK-021] Page numbers & headers/footers: implemented page-numbers.js with full header/footer tool under the Pages tab. Supports page number formats ("1", "Page 1", "Page 1 of N", Roman numerals), custom text with placeholders ({page}, {total}, {date}, {filename}), 6-position grid (top/bottom x left/center/right), font family (Helvetica/TimesRoman/Courier), font size (8-24pt), color picker, configurable margin, page range selection (all/skip first/custom), and live canvas preview. Uses pdf-lib drawText() for embedding. Output as -numbered.pdf. All 21 DOM IDs match, CSS styles defined, responsive layout. HTTP 200 from live site.
- [TASK-010] Digital signatures: implemented signatures.js (~550 lines) with three creation modes — draw on canvas (with color/thickness controls, touch support, auto-trim whitespace), type with 4 font choices (cursive/serif/sans/mono with live preview), and upload image (PNG/JPG/WebP with drag-and-drop). Signatures placed on PDF pages with drag-to-reposition and corner-resize handles. PDF embedding via pdf-lib embedPng/embedJpg with correct coordinate transformation. Integrated into main download flow in annotate.js. Output as -signed.pdf. Full keyboard accessibility (Delete to remove, focus indicators). Responsive CSS. All files serve HTTP 200.
